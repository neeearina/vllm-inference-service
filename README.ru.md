# vllm-inference-service

`vllm-inference-service` — сервис инференса на базе `FastAPI` и `vLLM`.
Сервис предоставляет HTTP API для генерации текста и возвращает:

- сгенерированный ответ;
- время обработки запроса (`latency_ms`).

Базовая модель в текущей конфигурации: `facebook/opt-125m` (см. `app/inference.py`).

## Запуск сервиса

Установка зависимостей:

```bash
pip install -r requirements.txt
```

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

После запуска сервис доступен по адресу: `http://127.0.0.1:8000`.

## API

### `POST /generate`

Назначение: генерация текста по входному промпту.

Пример запроса:

```json
{
  "prompt": "Explain what batching is",
  "max_tokens": 100,
  "temperature": 0.7
}
```

Пример ответа:

```json
{
  "response": "Generated text...",
  "latency_ms": 412.34
}
```

Ограничения валидации:

- `prompt`: непустая строка;
- `max_tokens`: от 1 до 512;
- `temperature`: от 0.0 до 2.0.

## Эксперименты

Перед запуском скриптов убедитесь, что API уже запущен на `127.0.0.1:8000`.

### 1) Warmup-эксперимент

Скрипт:

```bash
python scripts/warmup_experiment.py
```

Сценарий:

- 1 «холодный» запрос;
- 5 последовательных «тёплых» запросов с тем же телом.

Результаты:

| Метрика | Значение |
|---|---|
| Cold (1st request) | 627.6 ms |
| Warm #1 | 803.6 ms |
| Warm #2 | 961.4 ms |
| Warm #3 | 448.2 ms |
| Warm #4 | 936.8 ms |
| Warm #5 | 946.8 ms |
| Warm average (5) | 819.3 ms |
| Cold / Warm avg | 0.77x |

Первый запрос (`627.6 ms`) оказался быстрее среднего значения для пяти последующих запросов (`819.3 ms`). Полученный результат допустим для короткой серии наблюдений и отражает вариативность времени отклика между отдельными запросами.

### 2) Нагрузочный эксперимент

Скрипт:

```bash
python scripts/load_test.py
```

Сценарий: измерение средней задержки и пропускной способности при 1/5/10 одновременных запросах.

Результаты:

| Параллелизм | Средняя latency | Throughput | Успешно |
|---|---|---|---|
| 1 | 392.8 ms | 2.55 req/s | 1/1 |
| 5 | 2596.5 ms | 1.35 req/s | 5/5 |
| 10 | 3487.4 ms | 1.75 req/s | 10/10 |

С увеличением уровня параллелизма наблюдается существенный рост средней задержки (`392.8 ms` -> `2596.5 ms` -> `3487.4 ms`). Динамика соответствует сценарию ограниченной вычислительной емкости одного процесса инференса и формированию очереди запросов.
